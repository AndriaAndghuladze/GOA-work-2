Django-არის პითონის ფრეიმვორკი და გამოიყენება ბექენდიში

 django -> fullstack framework, რომელსაც შეუძლია უზრუნველყოს როგორც front end-ს ისე back end-ს, front end ის გამართვა ხდება template-ბის საშუალებით, როცა ვიყენებთ როგორც fullstack framework ს, ჩვენ გვაქ მხოლოდ 2 http მეთოდი, get & post, get ინფორმაციის წამოღებისას ბაზიდან, post ბაზის შეცვლასთან დაკავშირებული ნებისმიერი მოქმედება
 fullstack framework -> get, post

Django-კარგი რის დიდი და მძიმე ინფორმაციის დასამუშავებლად და ნაკლი ის არის რომ ამ მონაცემების დამუშავებას დიდი ხანი უნდება

django ს არქუტექტურა არის ერთ ერთი უადვილესი არქიტექტურა ყველა framework ს შორის, მას ეეწოდება mvt და იგი იყოფა 3 ნაწილად

 model -> აქ ჩვენ ვქმნით მონაცემთა ბაზის სტრუქტურად, სადაც ეგ ბაზა არის ობიექტად სახით წარმოდგენილი, ანუ ჩვენ არ გვიწევს SQL ის ან სხვა ბაზის კოდის წერა, django თავისით უზურნველყოფს მაგას, ჩვენგან ესაჭირეობა მხოლოდ მონაცემთა ბაზის სტრუქტურა

 view -> აქ ჩვენ ვამუშავებთ request ებს და ვაგზავნით response ებს, ადვილად რომ ვთქვათ აქ ვქმნით ფუნქციებს, რომელიც არგუმენტად იღეებს request ს და აბრუნებს response ს

 template -> თვითონ front end (როგორც ვთქვით front end ის გამართვა template ით ხდება მარტო როცა ვიყენებთ django ს როგორც full stack framework ს), ადდვილად რომ ვთ  ქვათ ეს არის html სადაც გამოდის ჩვენი ვებსაიტის შედეგი


ORM -> object relational mapping

ORM ნიშნავს რომ მონაცეემთა ბაზის კოდის წერა კონკრეტულად როცა ვმუშაობთ sql თან ჩვენ არ გვიწევს, მონაცემთა ბაზა და მისი შემადგენელი ნაწილები არის წარმოდგენილი ჯანგოში ობიექტების სახით, და ამ ობიექტებს აქვთ თავიანთი მეთოდები რომლებიც თვითონ უზრუნველყოფენ სხვადასხვა მოქდემებებს, როგორიცაა ბაზაში ჩამატება, ბაზიდან ამოშლა, ბაზის მონაცემების გაფლიტრვა, დასორტირება და ა.შ

ასევე django არის batteries-included framework, რადგან django ს აქვს ძალიან ბევრი ჩაშენებული ფუნქცია და მეთოდი რომლებიც გვაძლვენ იმის საშუალებას რომ დამატებითი ბიბლითოეკები და framework აღარ გამოვიყენოთ, რადგან ყველაფერი უკვე ჩაშეენებულია, მაგალითად user ის აუთენთიკაციის, login ინ და logout ის ფუნქციები უკვე ჩაშენეებული აქვს, ასევე აქვს email ზე ინფორმაციის გაგზავნის საშუალება ჩაშენებული, დარენდერების ფუნქცია ჩაშენებული და ა.შ


jango გამოიყენება დიდი და საშუალო ვებ-აპლიკაციებისთვის, როცა გჭირდება ავტორიზაცია, Admin Panel და ORM.

Django App არის პროექტის დამოუკიდებელი ფუნქციური ნაწილი.


Pipenv არის ინსტრუმენტი, რომელიც აერთიანებს pip-სა და Virtual Environment-ს.
